## 🧠 Engenharia de Prompts

Trabalhar com o **NotebookLM** trouxe aprendizados valiosos sobre como interagir com modelos de linguagem baseados em fontes fechadas.

### 🛠 Estratégias de Prompting

Testei variações para evitar respostas genéricas:

1.  **Prompt Direto:** "Resuma o livro de Wes McKinney." (Resultado muito superficial).
2.  **Prompt Estruturado:** "Com base na fonte 'Python para Análise de Dados', extraia os 5 métodos de limpeza de dados mais eficientes para grandes datasets e cite a página." (Resultado excelente).

### 🩹 Dificuldades Encontradas

  * **O Problema da Mistura:** Em temas complementares (ex: Pandas vs. Polars), a IA por vezes misturava sintaxes de implementações diferentes.
  * **A Solução:** A grande vantagem do NotebookLM é a **seleção de fontes**. Aprendi que, para obter respostas técnicas precisas, eu devia desmarcar as fontes de "tendências" e manter apenas os "manuais técnicos" antes de perguntar sobre código. Isso eliminou alucinações e restringiu o modelo ao que é puramente funcional.

| :--- | :--- |
| **Data Wrangling** | O processo de limpar e transformar dados brutos em um formato utilizável. |

| **EDA** | Exploratory Data Analysis (Análise Exploratória de Dados). |
